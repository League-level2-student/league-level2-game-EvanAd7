Connect Four

by: Evan Adami

This game was made for my level 2 project at The League of Amazing Programmers.
Click here to download and play!

1. Why I Made This Game

I decided to make Connect Four because it is a simple game that everyone enjoys. There are no complicated rules or disputes, just a straightforward set of instructions that can lead to a whole lot of fun.

2. What I Learned

From user testing I learned that sometimes there will be problems with your code that you might have overlooked. I also received feedback that led to changes with how the winning scenario should look at the end of my game.

3. Architecture of My Game

In my game, the Connect Four class is a runner class that runs the entire game. The Game Panel class manages the key detection and draws all of the graphics. The Checker class creates an object type that is a checker, and these checkers are the game pieces that are dropped into the Connect Four board. The Object Manager class manages the checkers, and creates an array list that holds all of the checkers and their positions and colors. Finally, the Check Winner class detects if a player has won the game by creating a row of four checkers, and uses the defined positions of the checkers to do so.

4. Technical Implementation

One particular aspect of my game that I am proud of is the win detection system. This part of the game detects if there are four checkers in a row anywhere on the board, horizontally, vertically, or diagonally. This part was quite challenging to create, but I was pleased with the product. I created this system by tracking the rows and columns that each checker was positioned in when it dropped, and then checked all the spaces around that checker for a checker with the same color. This process repeated until four checkers of the same color in a row were located.

5. Most Challenging Part

While the Connect Four win detection system was the part of the project that I was most proud of, it was also the most challenging part to create, especially checking if there were four checkers in a row diagonally. One of the difficulties I faced while coding this part was figuring out how to check both sides of a checker for checkers of the same color, and then to keep checking in that direction until a checker of a different color was detected, then switching directions. I overcame this difficulty by using a variable with a positive or negative value that switched values whenever the win detection switched directions.

6. Future Fixes and Updates

If I were to improve this game in the future, I would first try to create a more appealing menu or user interface. Then, I would switch the way you drop the checkers to a button press rather than a key press. Finally, I would try to create a single player version where the player plays against a computer rather than another player.
